# Tradelab Strategy Charter

> **저장 위치 (필수):** 백엔드·프론트엔드 두 저장소 루트 양쪽에 동일 사본 배치
> **목적:** 모든 전략·보조지표·시그널·봇 룰이 이 헌장을 위반하지 않는다는 단일 검증 기준
> **로드:** Claude Code 매 세션 시작 시 자동 읽기 (CLAUDE.md 와 동일 우선순위)
> **변경 절차:** 헌장 자체 변경은 본인 명시 승인 + 변경 이력 보존 필수

## I. 7 정보 차원 (필수 커버 영역)

모든 시그널·전략은 다음 7개 정보 차원의 **명시적 어느 하나**에 매핑되어야 한다.
각 차원은 서로 다른 정보를 측정하므로 단일 시그널이 여러 차원 동시에 잡으려 하면 결국 1~2개만 잘하고 나머지는 약함.

| # | 차원 | 측정 대상 | 표준 지표 |
|---|---|---|---|
| 1 | **모멘텀** | 가격이 어느 방향·얼마나 강하게 움직이나 | RSI, MACD histogram, ROC |
| 2 | **변동성** | 가격이 얼마나 흔들리나 | Bollinger Bands, ATR, BB width |
| 3 | **추세** | 추세의 강도·방향 | ADX, +DI/-DI, EMA Ribbon |
| 4 | **거래량/유동성** | 진짜 매수자가 있나 | Volume z-score, OBV, CVD (Cumulative Volume Delta) |
| 5 | **시장 구조** | 의미 있는 지지/저항 | Fibonacci, 추세선, Order Block, Liquidity Pool |
| 6 | **거시 컨텍스트** | 시장 전체 위험선호도 | DXY, SOFR-IORB, Fear&Greed, BTC 도미넌스 |
| 7 | **온체인** | 실제 코인 이동·고래 행동 | Exchange Netflow, Whale Alert, Stablecoin Supply, Coinbase Premium |

## II. 3 운영 규칙 (위반 금지)

### 규칙 1 — 차원 중복 금지

같은 차원에 속하는 지표를 여러 개 추가하지 않는다. 동어반복이며 다중공선성 함정.

**위반 예:**
- RSI(모멘텀) + Stochastic(모멘텀) 동시 사용 ❌
- BB(변동성) + Keltner Channel(변동성) 동시 사용 ❌
- ADX(추세) + MACD signal line(추세 일부) 동시 사용 ❌

**올바른 예:**
- RSI(모멘텀) + BB(변동성) + ADX(추세) — 3개 다른 차원 ✓
- MACD histogram(모멘텀의 모멘텀) + RSI(모멘텀)는 같은 차원이지만 측정 각도가 달라 예외 허용 (단 가중 합 시 카테고리 max 적용)

**실무 검증:**
새 지표 추가 PR 마다 다음 질문에 답변 필수.
- 어느 차원인가?
- 같은 차원에 이미 있는 지표는?
- 추가 의미는 (a) 다른 정보 측정 (b) 동일 정보 보강 (c) 동어반복 — 어느 쪽?
- (c) 면 거부.

### 규칙 2 — 백테스트 알파 검증 후 추가

지표·전략 추가는 "있어 보여서" 가 아니라 **백테스트로 알파를 입증한 후**에만.

**최소 검증 요건:**
- 365일 백테스트 (또는 데이터 가용 기간 최대)
- 표본 ≥ 100 시그널
- Wilson 95% 신뢰구간으로 baseline 대비 유의미 차이
- Out-of-sample 검증 (Purged K-fold 또는 walk-forward)
- Multiple comparison 보정 통과 (Benjamini-Hochberg FDR)

**알파 무 혹은 음수면 production 거부.**

### 규칙 3 — 단독 시그널 금지

추가 전략·지표는 기존 BBDX 트리거(NUM/PTN/BB)를 **강화 또는 약화하는 가중치**로만 작동. 단독 시그널 발행 X.

**왜:**
- 시그널 폭주 방지
- 사용자 의사결정 단순화 (BBDX 단일 진입점 + 보조 컨텍스트)
- false positive 감소 (다중 컨플루언스)

**구현 예:**
- 온체인 Exchange Netflow 음수 → BBDX LONG 신뢰도 +15% 가산 ✓
- 온체인 Exchange Netflow 음수 → 별도 LONG 시그널 발행 ❌

## III. 검증 시스템 — 7차원 커버리지 체크

새 전략·지표 PR 마다 **자동 검증 체크리스트**:

```
[ ] 1. 모멘텀 차원: 어떤 지표로 커버? (없으면 부족)
[ ] 2. 변동성 차원: 어떤 지표로 커버?
[ ] 3. 추세 차원: 어떤 지표로 커버?
[ ] 4. 거래량/유동성 차원: 어떤 지표로 커버?
[ ] 5. 시장 구조 차원: 어떤 지표로 커버?
[ ] 6. 거시 컨텍스트 차원: 어떤 지표로 커버?
[ ] 7. 온체인 차원: 어떤 지표로 커버?

부족 차원 → 표준 지표 자동 추천 + 추가 권고
중복 차원 → 규칙 1 위반, PR 거부 또는 수정
모든 차원 커버 → 규칙 1 통과
```

### 부족 차원 자동 보완 매트릭스

만약 새 전략이 7차원 중 일부만 커버하면, 다음 표준 지표로 빈 차원을 자동 채움:

| 부족 차원 | 자동 추가 표준 지표 | Tradelab 우선 구현 |
|---|---|---|
| 모멘텀 | RSI(14) | 이미 BBDX 에 포함 |
| 변동성 | BB(20, 2) | 이미 BBDX 에 포함 |
| 추세 | ADX(14) + EMA(50/200) | EMA Ribbon 추가 권고 |
| 거래량/유동성 | Volume z-score (EMA 50 baseline) | CVD 추가 검토 |
| 시장 구조 | Fibonacci + 추세선 | Wave Tracker 통합 |
| 거시 컨텍스트 | Fear&Greed + DXY | Macro Liquidity Tracker 통합 |
| 온체인 | Exchange Netflow + Whale Alert | ONCHAIN_INTEGRATION.md 참조 |

## IV. 가중치 도출 원칙

7차원 합산 시 각 차원의 가중치는 **calibration 결과로 도출**. 직관 X.

```
finalScore = Σ (categoryScore_i × weight_i)
  단, Σ weight_i = 1.0
  단, weight_i = calibration 결과 (constrained least squares, R² 검증)
```

calibration 미완료 시 **균등 가중 (1/7)** 으로 시작 + UI 에 "베타: 가중치 미캘리브레이션" 명시.

## V. 자본 보호 우선

모든 전략은 다음 보호 룰을 위반할 수 없음:

- 거래당 max risk: 자본의 1%
- 단일 포지션 max: 자본의 5%
- 일일 손실 한도: 자본의 3% (도달 시 24시간 새 진입 차단)
- Circuit breaker: 시장 변동성·상관·유동성 3축 트리거 시 모든 시그널 30분 정지
- 실거래 봇: 30일 dry-run 후에만 활성화 가능

## VI. 헌장 위반 처리

PR 또는 코드 변경이 헌장 위반 시:

1. CI/CD 파이프라인이 자동 거부
2. PR 코멘트에 위반 항목 명시 (규칙 1, 2, 또는 3)
3. 본인 명시 승인 + 변경 이력 보존 후에만 통과 가능
